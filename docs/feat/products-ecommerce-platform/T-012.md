## Description

Create database migration and seed data to ensure required roles exist: Admin, Manager, Customer, Guest. Use ABP's DataSeeder or IDataSeedContributor to insert roles if not present. Ensure role names match permission system expectations.

---

## Acceptance

- Roles are created in database during initial migration or seeding
- Role names exactly: "Admin", "Manager", "Customer", "Guest"
- Seeding logic is idempotent (safe to run multiple times)
- Permissions can be assigned to these roles
- System starts with roles present
