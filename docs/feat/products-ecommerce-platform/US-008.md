## Title

[Products E-commerce Platform] Add user story and scenarios for admin user management and role assignment

---

## User Story

As an Admin,
I want to view, create, update, and manage system users with roles and permissions,
So that I can control access, onboard new staff, and maintain system security.

---

## Scenario 1: Successful flow

Given I am an authenticated Admin with full system access
When I create a new Manager user with email, password, role assignment (Manager), and profile data (FirstName, LastName)
Then a new IdentityUser is created with the specified role
And the corresponding Customer/User entity is created with appropriate role metadata
And I receive the created UserDto with UserId, Email, Role, and profile fields
And an audit log records the user creation event with AdminUserId
And the response status is 201 Created

---

## Scenario 2: Validation / error flow

Given I am an authenticated Admin
When I attempt to assign an invalid role (e.g., "SuperAdmin" that does not exist in the system)
Then I receive a 400 Bad Request response with validation error: "Role 'SuperAdmin' is not valid"
And the user creation or update does not proceed
And the error message lists available roles (Admin, Manager, Customer, Guest) without exposing system internals

---

## Scenario 3: Alternate flow

Given I am an authenticated Admin viewing the user list
When I search for users by partial email matching ("@example.com")
Then I receive a filtered list of users whose emails contain that pattern
The results include only active users by default (exclude soft-deleted unless searching all)
Pagination and sorting still apply to the filtered result set
And the response shows total matching count

---

## Acceptance Criteria

- Admin can view all system users (list with pagination, filtering, sorting)
- Admin can create new users with roles: Admin, Manager, Customer, Guest (enforced by permission system)
- Admin can update user profile data, role assignments, and active status (enable/disable, not delete)
- Admin can soft-delete users (set IsDeleted = true) but not hard-delete (for audit)
- Role changes are audited with who changed what and when
- Email uniqueness enforced across all users
- User list filters include: Email (contains), Role, IsActive, IsDeleted, CreatedDate range
- User details endpoint shows: UserId, Email, Roles, Profile (FirstName, LastName, PhoneNumber), LastLoginTime, IsActive, CreationTime
- Admin cannot modify other Admins' roles unless explicitly allowed by business rule (self-protection)
- The system logs all admin actions (user creation, role changes, deactivations) for compliance
- Permission checks: only users with Admin role can access these endpoints (ABP permission system)
- All uses ABP's IdentityUser integration with proper UserManager operations
