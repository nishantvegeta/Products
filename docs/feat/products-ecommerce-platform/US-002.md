## Title

[Products E-commerce Platform] Add user story and scenarios for customer registration and account creation

---

## User Story

As a Guest,
I want to register for a customer account with email and password,
So that I can access personalized features like address management and order placement.

---

## Scenario 1: Successful flow

Given I am an unauthenticated guest
When I submit a registration request with valid email (unique), strong password, and required profile fields (FirstName, LastName)
Then a new Customer user account is created with Active status
And I receive an authentication token (JWT) for immediate login
And the Customer entity is linked to the underlying IdentityUser
And the response status is 201 Created

---

## Scenario 2: Validation / error flow

Given I am an unauthenticated guest
When I submit a registration request with an email that already exists in the system
Then I receive a 400 Bad Request response with a validation error: "Email is already registered"
And no new user account is created
And the system does not expose whether the email exists in error messages (use generic message for security)

---

## Scenario 3: Alternate flow

Given I am an unauthenticated guest
When I submit a registration request with all valid fields but without setting a default address
Then a new Customer user account is created successfully
And the Customer entity has no addresses initially
And I can add addresses later through the address management API
And the response returns the created CustomerDto without address data

---

## Acceptance Criteria

- The API endpoint accepts Email, Password (hashed), FirstName, LastName, and optional fields (PhoneNumber)
- Email must be unique across all users (checked at registration)
- Password must meet complexity requirements (minimum length, uppercase, lowercase, digit, special char)
- Duplicate email attempts return 400 with non-revealing error message (security)
- Successful registration returns JWT token in response body for immediate authentication
- Customer entity is created with Customer role assigned
- The endpoint uses ABP's built-in IdentityUser integration for user management
- All input fields undergo proper validation (required, format, length)
- The system logs the registration event with timestamp and IP address (audit)
- Email verification workflow is out of scope for Phase 1 (verification flag defaults to true)
