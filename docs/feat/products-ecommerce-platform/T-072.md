# T-072 — Implement data masking for privacy

**Feature:** Products E-commerce Platform
**Layer:** Application Service
**Linked Story:** US-009
**Estimate:** 1h
**Status:** Open

---

## Description

In reporting and list endpoints, implement data masking for sensitive fields: customer emails and phone numbers may be masked (e.g., show only first letter or asterisks) in admin exports and reports unless explicitly requested unmasked. Provide parameter to include unmasked data for authorized admin actions.

---

## Acceptance

- Reporting and export DTOs mask email and phone by default
- Masking logic in AppService or DTO mapping
- Optional parameter (includeSensitive=true) allows unmasked data for Admin
- Masked format: email becomes "u***@example.com", phone becomes "***-***-1234"
- Audit logging tracks when sensitive data is accessed
