## Description

Extend CategoryAppService with full CRUD: CreateCategoryAsync (with ParentCategoryId support), UpdateCategoryAsync, DeleteCategoryAsync (soft with validation), GetCategoryAsync, GetCategoryListAsync. Implement deletion validation: cannot delete category that has active products (business rule BR-006 interpretation).

---

## Acceptance

- CategoryAppService CRUD methods implemented
- Create allows ParentCategoryId to be null or reference existing category
- Update handles ParentCategoryId changes (avoid circular references)
- Delete method checks for active products before soft delete (or defers to database FK depending on decision)
- Returns CategoryDto with ParentCategoryName for convenience
- Requires Manager permission
