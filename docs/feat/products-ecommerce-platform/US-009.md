## Title

[Products E-commerce Platform] Add user story and scenarios for admin to view all orders, products, customers, and system reports

---

## User Story

As an Admin,
I want to view all system data (orders, products, customers, categories) with comprehensive reporting capabilities,
So that I can monitor business operations, analyze trends, and make informed decisions.

---

## Scenario 1: Successful flow

Given I am an authenticated Admin
When I request a system overview with aggregated metrics (total orders, total revenue, active customers, product count)
Then I receive a DashboardSummaryDto with:
- TotalOrderCount
- TotalRevenue (sum of completed orders' TotalAmount)
- ActiveCustomerCount (non-deleted)
- ActiveProductCount
- RecentOrders (last 10 by date)
- TopSellingProducts (by quantity sold in last 30 days)
All metrics are calculated efficiently with proper database queries
And the response status is 200 OK

---

## Scenario 2: Validation / error flow

Given I am an authenticated Admin
When I attempt to access an admin-only endpoint without proper permission (role other than Admin)
Then I receive a 403 Forbidden response
And the response body indicates insufficient permissions
And the operation is not executed

---

## Scenario 3: Alternate flow

Given I am an authenticated Admin
When I export all orders data with filters (date range, status) in CSV format
Then I receive a CSV file containing selected order fields: OrderId, OrderNumber, CustomerEmail, OrderDate, Status, TotalAmount
The file is generated server-side with proper streaming for large datasets
The response includes Content-Disposition header for file download
And the export operation is logged in the audit trail

---

## Acceptance Criteria

- Admin has unrestricted read access to all entities: Orders (all customers), Products, Categories, Customers, User data
- Admin can view detailed reports: sales by date range, product popularity, customer order counts, revenue trends
- Reporting endpoints support date range filtering, grouping (by day, month, product, category), and aggregation
- Dashboard summary API returns key business metrics (KPIs) with reasonable performance (<2s response)
- Admin can export data (CSV, Excel) for orders, products, customers for external analysis
- All admin reporting endpoints enforce Admin role permission checks
- The system uses optimized queries with proper indexes for large datasets (avoid full table scans)
- Sensitive data (customer emails, phone numbers) is masked or excluded from public-facing exports based on privacy rules
- Audit logging: all admin data access to customer records is logged for compliance (GDPR/CCPA considerations)
- The endpoints follow ABP's permission system: define permissions like "Admin.Dashboard.View", "Admin.Reports.View"
- Pagination is enforced for list endpoints to prevent memory issues (max page size 100 for admin, potentially higher with special permissions)
- Data consistency: reports show current data including pending transactions (or specify snapshot policy)
