## Title

[Products E-commerce Platform] Add user story and scenarios for customer profile and address management

---

## User Story

As a Customer,
I want to view and update my profile information and manage multiple addresses with designated types,
So that I can maintain accurate personal data and use different billing/shipping addresses for orders.

---

## Scenario 1: Successful flow

Given I am an authenticated Customer
When I update my profile with new FirstName, LastName, and PhoneNumber
Then my profile data is updated successfully
And I receive the updated CustomerDto with the new values
And the LastModificationTime is set to the current timestamp
And the response status is 200 OK

---

## Scenario 2: Validation / error flow

Given I am an authenticated Customer
When I attempt to add a new address with missing required fields (e.g., Street, City, Country)
Then I receive a 400 Bad Request response with validation errors listing the missing fields
And the address is not created
And my existing addresses remain unchanged

---

## Scenario 3: Alternate flow

Given I am an authenticated Customer with existing addresses
When I add a new address with AddressType = "Shipping" and set IsDefault = true
Then the new address is created with IsDefaultShipping = true
And all my other addresses with AddressType = "Shipping" have IsDefaultShipping set to false (only one default per type)
And the response returns the created AddressDto with IsDefaultShipping = true
And the AddressId is generated and returned

---

## Acceptance Criteria

- authenticated Customer can view their own profile via GET /api/app/customer/profile
- Customer can update their own profile via PUT /api/app/customer/profile (excluding email, password)
- Customer can add new addresses with fields: Street, City, State, PostalCode, Country, AddressType (Billing/Shipping), IsDefault (boolean)
- Customer can list all their addresses (GET /api/app/customer/addresses)
- Customer can update or delete their own addresses (PUT/DELETE with address ID)
- Only one default address per AddressType is allowed (setting new default removes it from others)
- Address operations are scoped to the Customer's own data (cannot access other customers' addresses)
- All monetary calculations use decimal type with USD currency
- Soft delete is used for addresses (IsDeleted flag)
- Address queries return data ordered by IsDefault descending, then creation time
