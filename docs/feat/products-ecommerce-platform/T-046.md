# T-046 — Define Product/Category DTOs with validation

**Feature:** Products E-commerce Platform
**Layer:** Validation
**Linked Story:** US-006
**Estimate:** 1h

## Description

Create DTO classes and define validation rules: CreateProductDto requires Name, SKU, Description, Price (>0), StockQuantity (≥0), CategoryId. UpdateProductDto similar but optional fields. Category DTOs validate Name required, DisplayOrder optional. Use DataAnnotations or FluentValidation.

---

## Acceptance

- CreateProductDto with required fields and validation attributes
- UpdateProductDto with optional fields where appropriate
- Name max length, SKU max length constraints
- Price validated as positive decimal
- CategoryId validates existing category (server-side check in service)
- Error messages localized if needed
