## Title

[Products E-commerce Platform] Add user story and scenarios for product and category CRUD with hierarchical categorization

---

## User Story

As a Manager,
I want to create, update, and manage products and hierarchical categories,
So that I can maintain the product catalog with proper organization and inventory information.

---

## Scenario 1: Successful flow

Given I am an authenticated Manager with product management permission
When I create a new Product with required fields: Name, SKU (unique), Description, Price (decimal), StockQuantity, CategoryId
Then the Product is created with active status
And the SKU is validated to be unique across all products
And the CategoryId references an existing (non-deleted) category
And the response returns the created ProductDto with generated Id and timestamps
And the response status is 201 Created

---

## Scenario 2: Validation / error flow

Given I am an authenticated Manager
When I attempt to update Product.StockQuantity to a negative value
Then I receive a 400 Bad Request response with validation error: "StockQuantity must be zero or greater"
And the Product remains unchanged
And the validation error includes the field name and constraint message

---

## Scenario 3: Alternate flow

Given I am an authenticated Manager managing categories
When I create a subcategory with ParentCategoryId pointing to an existing category
Then the Category is created with ParentCategoryId set
And the hierarchical structure allows infinite nesting depth
And I can retrieve the full category tree via the categories API
And the response returns the new CategoryDto with ParentCategoryName populated (if desired)

---

## Acceptance Criteria

- Manager can perform full CRUD on Products: Create, Read, Update, (soft) Delete
- Manager can perform full CRUD on Categories: Create, Read, Update, (soft) Delete with ParentCategoryId support
- Product entity includes: Id, Name, SKU (unique), Description, Price (decimal), StockQuantity (≥0), CategoryId (FK), IsActive (bool), IsDeleted
- Category entity includes: Id, Name, Description, ParentCategoryId (nullable FK to self), DisplayOrder (optional), IsDeleted
- Unique constraint on Product.SKU enforced at database level
- Foreign key constraints: Product.CategoryId → Category.Id, Category.ParentCategoryId → Category.Id
- Category deletion validation: cannot delete category that has active products (referential integrity or business rule)
- Product and Category updates use proper DTOs (CreateProductDto, UpdateProductDto, ProductDto, etc.)
- All updates audit CreationTime, CreatorId, LastModificationTime, LastModifierId automatically via ABP
- Pagination and filtering (by name, SKU, category) supported on Product and Category list endpoints
- Soft delete implemented: IsDeleted = true moves to recycle bin, queries filter by IsDeleted = false by default
- Managers have permission to manage products/categories but not other managers' data unless specified
- The system enforces BR-001: unique SKUs; BR-006: soft delete preference; BR-008: decimal monetary fields
