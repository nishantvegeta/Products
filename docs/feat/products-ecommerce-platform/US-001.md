## Title

[Products E-commerce Platform] Add user story and scenarios for browsing product catalog as guest

---

## User Story

As a Guest,
I want to browse the product catalog with hierarchical categories,
So that I can discover products and decide whether to place an order.

---

## Scenario 1: Successful flow

Given I am an unauthenticated guest user
When I request the product catalog with pagination parameters (page 1, page size 20)
Then I receive a paginated list of active products with basic details (name, SKU, price, stock status)
And I receive the hierarchical category structure with parent-child relationships
And each product includes its assigned category information
And the response status is 200 OK

---

## Scenario 2: Validation / error flow

Given I am an unauthenticated guest user
When I request the product catalog with an invalid page size (e.g., 200, exceeding the maximum of 100)
Then I receive a 400 Bad Request response with a validation error message
And the product catalog is not returned

---

## Scenario 3: Alternate flow

Given I am an unauthenticated guest user
When I request products filtered by a specific category (e.g., "Electronics")
Then I receive only products belonging to that category and its subcategories
And the category hierarchy still returns the full structure for navigation
And the response includes pagination metadata (total count, current page)

---

## Acceptance Criteria

- The API endpoint returns active products without authentication requirements
- Pagination is supported with configurable page size (max 100 items per page)
- The product response includes at least: Id, Name, SKU, Price (decimal, USD), StockQuantity, and CategoryId
- Categories include hierarchical parent-child relationships with unlimited nesting depth
- Products include category references (CategoryId, CategoryName) for display
- Invalid pagination requests return 400 with clear error details
- The system properly handles empty catalog (no products) by returning empty results with pagination metadata
- API response follows ABP's standard wrapper format: `{ "result": {...}, "target": "...", "success": true }`
