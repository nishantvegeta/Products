## User Story

As a customer,
I want to create, view, update, delete, and set default billing and shipping addresses,
So that I can reuse my addresses for orders instead of re-entering information each time.

---

## Scenario 1: Successful flow

Given I am a logged-in customer with no saved addresses
When I create a new address with all required fields (street, city, state, postal code, country) and mark it as default billing
Then the address is saved successfully
And I can view it in my address list
And it is marked as the default billing address

---

## Scenario 2: Validation / error flow

Given I am a logged-in customer
When I attempt to create an address with missing required fields (e.g., no postal code)
Then the system returns a validation error
And the address is not saved
And I remain on the address creation form with error messages displayed

---

## Scenario 3: Alternate flow

Given I am a logged-in customer with two existing addresses (one marked as default billing)
When I create a new address and mark it as default billing
Then the new address becomes the default billing address
And the previous default billing address is automatically unmarked as default
And both addresses remain in my address list

---

## Acceptance Criteria

The customer can add a new address with all required fields.
The customer can view a list of all their saved addresses.
The customer can edit an existing address.
The customer can delete an address.
The customer can designate one address as default billing.
The customer can designate one address as default shipping.
The same address can be both default billing and default shipping.
Only one address can be set as default billing at a time.
Only one address can be set as default shipping at a time.
Customers can only access their own addresses.
