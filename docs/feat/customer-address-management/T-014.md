# T-014 — Add Admin Endpoints to Address API Controller

**Feature:** Customer Address Management
**Layer:** API / Controller
**Linked Story:** US-002
**Estimate:** 1h

---

## Description

Extend AddressController with admin-specific endpoints for cross-customer address management. Use separate routes or query parameters to distinguish admin operations from customer operations.

---

## Acceptance

- GET /api/app/admin/addresses/customer/{customerId} — returns all addresses for a specific customer
- GET /api/app/admin/addresses/search — supports filtering by customer name/ID, address type, default status
- DELETE /api/app/admin/addresses/{id}?reason={reason} — admin delete with reason parameter
- All admin endpoints require admin permission via [RequiredPermission] attribute
- Admin endpoints bypass customer ownership checks but still respect soft-delete
- Responses follow standard ABP format
