# T-014 — Add Admin Endpoints to Address API Controller

**Feature:** Customer Address Management
**Layer:** API / Controller
**Linked Story:** US-004
**Estimate:** 1h
**Status:** Open

---

## Description

Build a dedicated API controller (or extend existing admin controller) to expose administrative address management endpoints. These endpoints allow admins to query addresses by customer, view full details, and perform corrective updates or deletions. Implement appropriate permission checks and audit logging.

---

## Acceptance

- AdminAddressController (or similar) with endpoints: GET /api/admin/addresses/by-customer/{customerId}, GET /api/admin/addresses/{id}, PUT, DELETE.
- All endpoints require admin permission (e.g., AddressManagement.AdministratorAccess).
- Controller injects admin address service and handles request/response mapping.
- Audit logging is triggered on all write operations (update, delete).
- Responses follow standard API format.
- API documentation includes admin endpoints with proper descriptions and permission notes.
