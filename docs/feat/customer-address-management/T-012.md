# T-012 — Modify Order Creation Logic to Accept Address Selection

**Feature:** Customer Address Management
**Layer:** Application Service
**Linked Story:** US-003
**Estimate:** 2h
**Status:** Open

---

## Description

Enhance the order creation process to accept customer-selected saved addresses or new ad-hoc address entries. Update the OrderAppService (or equivalent) to handle the address selection logic, including validation that at least one address is provided, default usage when applicable, and optional saving of new addresses during checkout.

---

## Acceptance

- Order creation method (e.g., CreateAsync) accepts parameters for billing and shipping address selection (either saved address Ids or full address data).
- Validation ensures both billing and shipping addresses are provided before order is created.
- When a saved address is selected, it is validated to belong to the customer (ownership check).
- If a new address is entered during checkout, the system optionally creates a new Address entity and links it (based on user's save preference).
- Default address fallback logic: if no specific address is chosen, use the customer's default billing/shipping addresses if available.
- Proper error messages returned for invalid scenarios (e.g., address not found, wrong owner).
- Unit tests cover: saved address selection, ad-hoc address entry with save, default fallback, validation failures.
