# T-022 — Build Admin Interface for Customer Address Support

**Feature:** Customer Address Management
**Layer:** Frontend — Page / View
**Linked Story:** US-004
**Estimate:** 2h
**Status:** Open

---

## Description

Create an administrative page that allows system administrators to search for customers and view/manage their saved addresses. Include customer search functionality, address list display, and actions to edit or delete addresses. Ensure the page is accessible only to users with admin role and uses admin-specific API endpoints.

---

## Acceptance

- Admin page route: /admin/customers/{customerId}/addresses or an address management section in admin panel.
- Admin can search for customers by ID, name, or email.
- Customer search results lead to address list view for selected customer.
- Address list shows all addresses with full details and default status.
- Admin can edit or delete customer addresses (with confirmation).
- All admin actions are logged on the server side.
- Page checks admin permission before rendering or allowing actions.
- Loading and error states handled; user feedback provided.
