# T-005 — Implement AddressAppService Business Logic

**Feature:** Customer Address Management
**Layer:** Application Service
**Linked Story:** US-001
**Estimate:** 2h

---

## Description

Implement the AddressAppService with full CRUD logic, including default address management (only one default billing and one default shipping per customer). The service uses repository pattern, mapper for DTO conversions, and handles business rules.

---

## Acceptance

- CreateAsync validates input, sets CustomerId from current user, handles default flags (unset other defaults if needed), saves entity
- UpdateAsync validates, updates fields, handles default flag changes (unset other defaults if needed), returns updated address
- DeleteAsync performs soft delete (sets IsDeleted), respects ABP soft-delete pattern
- GetAsync retrieves single address by Id with customer ownership check
- GetListAsync returns paginated list of addresses for current customer
- Business rules enforced: BR-003, BR-004, BR-005 (default uniqueness)
- Repository used for data access via IRepository<Address, Guid>
- IObjectMapper used for entity-DTO mapping
- CurrentCustomerId retrieved from ICurrentUser (ABP) or equivalent
