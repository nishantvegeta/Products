# T-019 — Build Address Form Component with Validation

**Feature:** Customer Address Management
**Layer:** Frontend — Component
**Linked Story:** US-001, US-002
**Estimate:** 2h
**Status:** Open

---

## Description

Create a reusable form component for creating and editing addresses. The form should include fields for all required address properties (street, city, state, postal code, country) and checkboxes for setting as default billing/shipping. Implement client-side validation that mirrors server-side rules and provides clear error messages.

---

## Acceptance

- Form component accepts address data (for edit) and a mode (create/edit).
- Input fields: Street (text), City (text), State/Province (text or dropdown), PostalCode (text), Country (text or dropdown).
- Checkboxes: "Set as default billing", "Set as default shipping".
- Required field validation displays inline errors if fields are empty.
- Submit button disabled until valid data entered.
- On submit, calls the appropriate backend API (create or update).
- Success/error feedback shown to user (e.g., toast notifications).
- Component integrates with the address management page.
- Form resets or navigates away after successful submission.
