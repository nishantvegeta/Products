# T-013 — Implement Admin Methods in AddressAppService

**Feature:** Customer Address Management
**Layer:** Application Service
**Linked Story:** US-004
**Estimate:** 1.5h
**Status:** Open

---

## Description

Extend the address management service or create a separate admin service to allow administrators to bypass customer ownership restrictions. Implement methods for admins to retrieve, update, and delete any customer's addresses. Add logging for all admin actions on customer addresses for audit trail.

---

## Acceptance

- New admin-specific methods or a service class exists for administrative address access.
- Admin methods accept a customerId parameter to target the correct customer's addresses.
- Permission checks enforce that only users with Administrator role can call these methods.
- All admin actions are logged with: admin user ID, target customer ID, address ID, action type, timestamp.
- Admin operations use the same validation and business rules as regular operations, except ownership is overridden.
- Unit tests confirm admin can access any customer's addresses and regular users cannot.
