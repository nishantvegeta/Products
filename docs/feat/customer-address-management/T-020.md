# T-020 — Integrate Address Selection into Order Checkout Page

**Feature:** Customer Address Management
**Layer:** Frontend — Component
**Linked Story:** US-003
**Estimate:** 2.5h
**Status:** Open

---

## Description

Modify the order creation/checkout page to allow customers to select saved billing and shipping addresses. Add UI components that display the customer's address book and allow selection via radio buttons or dropdowns. Provide alternate option to enter new address manually. Integrate with the order API to send selected address IDs or new address data.

---

## Acceptance

- Checkout page shows sections for Billing Address and Shipping Address.
- Each section has two options: "Select from my addresses" (displays list/tile of saved addresses) or "Enter a new address" (shows address form).
- Default addresses are pre-selected if they exist, otherwise no selection is made.
- When a saved address is selected, the address details are displayed (read-only) for confirmation.
- New address form includes a "Save this address for future use" checkbox.
- Both billing and shipping must have an address before order submission is allowed.
- Validation ensures addresses are provided; error messages shown if missing.
- On order submission, the selected address IDs (or new address data) are sent to the order API.
- Loading states and error handling are implemented.
