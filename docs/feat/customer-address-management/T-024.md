# T-024 — Handle Default Address Constraint in Deletion Workflow

**Feature:** Customer Address Management
**Layer:** Application Service
**Linked Story:** US-001, US-002
**Estimate:** 1h
**Status:** Open

---

## Description

Implement the logic to prevent deletion of an address that is currently set as default billing or shipping, or to automatically clear the default flag if the address is deleted. Decide and implement the behavior based on the open question: require customer to change default first, or clear default automatically upon deletion.

---

## Acceptance

- DeleteAddressAsync method checks if the address being deleted is a default (billing or shipping).
- If configured to prevent deletion: method throws an exception with a clear message prompting user to set a new default first.
- If configured to auto-clear: method clears IsDefaultBilling and/or IsDefaultShipping before deletion.
- Appropriate error is surfaced to the frontend with user-friendly message.
- Unit tests cover both scenarios (delete default vs. delete non-default) based on chosen approach.
- Business rule BR-009 is satisfied.
