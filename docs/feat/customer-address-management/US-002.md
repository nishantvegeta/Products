## Title

[Customer Address Management] Add user story and scenarios for designating default billing and shipping addresses

---

## User Story

As a customer,
I want to designate one of my saved addresses as the default billing address and one as the default shipping address,
So that I can speed up order checkout by having pre-selected defaults.

---

## Scenario 1: Successful flow

Given I have multiple saved addresses in my address book
When I select one address as "Set as Default Billing" and another (or the same) as "Set as Default Shipping"
Then the system marks those addresses as the respective defaults
And the billing and shipping defaults are clearly indicated in my address list
And only one address can be set as default billing and only one as default shipping at any time

---

## Scenario 2: Validation / error flow

Given I have only one saved address
When I attempt to delete that address while it is still set as my default billing or shipping address
Then the system prevents deletion or prompts me to first select a different default address
And my address remains unchanged

---

## Scenario 3: Alternate flow

Given I have an address currently set as my default billing address
When I select a different address as the new default billing
Then the previous default billing address loses its default status
And only the newly selected address is marked as default billing

---

## Acceptance Criteria

- The customer can designate any saved address as the default billing address.
- The customer can designate any saved address as the default shipping address.
- The system enforces that at most one address can be the default billing address at a time.
- The system enforces that at most one address can be the default shipping address at a time.
- The same address can be set as both default billing and default shipping.
- Default status is clearly visible in the address list UI.
- Changing a default automatically clears the previous default of that type.
- The customer cannot remove the default designation from an address without either setting a new default or deleting the address (subject to constraints).
