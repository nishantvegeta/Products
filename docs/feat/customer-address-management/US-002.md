## Title

[Feature] Add user story and scenarios for Customer Address Management — View and Manage All Customer Addresses

---

## User Story

As a system administrator,
I want to view and manage all customer addresses in the system,
So that I can assist customers with address issues and perform support tasks.

---

## Scenario 1: Successful flow

Given I am logged in as a system administrator with admin privileges
When I search for a customer by name or ID
Then I can view all addresses associated with that customer
And I can see address details including owner, creation date, and default status
And I can perform administrative actions (view, soft-delete, modify if needed)

---

## Scenario 2: Validation / error flow

Given I am logged in as a system administrator
When I search for a customer with no saved addresses
Then the system displays a message indicating no addresses exist
And I am not shown an empty address list

---

## Scenario 3: Alternate flow

Given I am logged in as a system administrator
When I attempt to modify or delete an address
Then I must provide a reason for the change (audit trail requirement)
And the action is recorded in the system audit log
And the affected customer is notified of the change (if applicable)

---

## Acceptance Criteria

Administrators can view all addresses belonging to any customer.
Administrators can search and filter addresses by customer, address type, or default status.
Administrators can view full address details.
Administrators can soft-delete addresses (respecting ABP soft-delete pattern).
All administrative actions on addresses are logged in the audit trail.
Administrator access is enforced via permission checks.
Customer data isolation is bypassed for administrators (they see all customers).
