# T-016 — Write Integration Tests for Admin Endpoints

**Feature:** Customer Address Management
**Layer:** Testing

## Description

Write integration tests for admin API endpoints, testing both permission enforcement and functional behavior. Use both admin user and non-admin (customer) test accounts.

---

## Acceptance

- Tests for admin GET /addresses/customer/{customerId} with admin user returns addresses
- Tests for admin endpoints with customer user returns 403 Forbidden
- Tests for admin search with various filters returns correct results
- Tests for admin delete with reason parameter performs soft delete
- Tests confirm audit log entries are created (verify via test double or database check)
- Tests verify notifications are sent to affected customer on admin modifications
- Database seeded with customers having addresses for test scenarios
