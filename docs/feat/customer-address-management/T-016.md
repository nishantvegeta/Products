# T-016 — Enhance Order Read Model to Include Address Details

**Feature:** Customer Address Management
**Layer:** Application Service
**Linked Story:** US-005
**Estimate:** 1h
**Status:** Open

---

## Description

Update order query operations (GetOrder, GetOrderList, etc.) to include both saved address references and ad-hoc address details. Ensure that order processors can see the complete billing and shipping information regardless of whether the address is saved or one-time. Consider whether to show current saved address data or preserve historical snapshot based on open question.

---

## Acceptance

- Order queries return address information: either linked AddressId with populated address fields, or explicit billing/shipping fields for one-time addresses.
- For orders using saved addresses, the response includes sufficient address detail to fulfill the order (all fields: street, city, state, postal code, country).
- Implementation decision on historical snapshot vs. current data is documented and implemented (likely preserve snapshot by copying address fields to order at creation time).
- Unit tests verify order queries return complete address details for both saved and one-time address orders.
- Order processor acceptance criteria are met regarding display and access.
