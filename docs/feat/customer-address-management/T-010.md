**Layer:** Data / Entity

## Description

Modify the Order entity to store references to saved addresses when a customer selects them during checkout. Add foreign key properties for BillingAddressId and ShippingAddressId, both nullable to support one-time addresses. Establish relationships to the Address entity while preserving order history integrity.

---

## Acceptance

- Order entity gains nullable properties: BillingAddressId, ShippingAddressId (type Guid? or matching Address Id type).
- Navigation properties: BillingAddress, ShippingAddress to the Address entity are defined.
- Relationship configuration in DbContext ensures orders maintain references even if addresses are later deleted (consider setting DeleteBehavior.Restrict or nullify).
- Migrations are created to add these fields to the orders table.
- Unit tests verify that an Order can reference an Address entity.
- Backward compatibility: existing orders without address references continue to work (null values allowed).
