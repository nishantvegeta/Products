## Description

Build integration tests exercising the API endpoints end-to-end with a test database (e.g., SQLite in-memory or test PostgreSQL). Test all CRUD operations, permission checks, default address behavior, and order creation with address selection. Simulate multiple users and roles (customer, admin, order processor) to verify access controls.

---

## Acceptance

- Integration test project (or existing project) includes AddressManagementIntegrationTest class.
- Tests cover: create/read/update/delete addresses as customer; default setting behavior; order creation with saved address; order creation with ad-hoc address; admin access to any customer's addresses; order processor read access.
- Tests use test data builder pattern or fixtures to set up customers and addresses.
- Database is reset between tests or tests are isolated.
- Assertions validate HTTP status codes, response content, and database state.
- Permission violations are tested: customer cannot access other customer's addresses, non-admin cannot use admin endpoints.
- All integration tests pass.
