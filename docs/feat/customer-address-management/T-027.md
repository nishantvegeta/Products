**Layer:** Config / Migration

## Description

Update the project's permission and role seeding logic to include the new address management permissions. Ensure that suitable roles (Admin, Customer, OrderProcessor) have appropriate access levels according to the user stories. This may involve modifying a SeedData class or database migration that sets up initial permission data.

---

## Acceptance

- New permission names are added to the permission definition provider (if using ABP Permission Management).
- Role permission data includes:
  - Admin role: all address permissions.
  - Customer role: create/read/update/delete own addresses, set defaults.
  - OrderProcessor role: view addresses (read-only).
- Seed data runs successfully on new database installations.
- Existing databases can be updated via migration or manual permission assignment.
- Internationalization/localization for permission names if used in UI.
