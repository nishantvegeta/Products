# T-007 — Implement Address AppService with Business Logic

**Feature:** Customer Address Management
**Layer:** Application Service
**Linked Story:** US-001, US-002
**Estimate:** 2h
**Status:** Open

---

## Description

Create the AddressAppService class that orchestrates address CRUD operations and enforces business rules. Implement default address logic that ensures only one default billing and one default shipping address per customer. Use the repository pattern and inject required dependencies (repository, mapper, permission checker). Follow ABP's ApplicationService base class conventions.

---

## Acceptance

- AddressAppService inherits from ApplicationService (or CrudAppService if appropriate).
- Methods: CreateAsync, UpdateAsync, DeleteAsync, GetAsync, GetAllAsync (with pagination if required).
- Default address logic: when setting a new default, automatically clears previous default of the same type within the same customer.
- Proper validation of required fields and ownership checks.
- Exceptions are thrown with appropriate error messages for invalid operations.
- Methods return DTOs or PagedResultDto as appropriate.
- Unit tests cover all business logic scenarios including edge cases.
