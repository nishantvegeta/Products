# T-021 — Implement Frontend State Management for Address Data

**Feature:** Customer Address Management
**Layer:** Frontend — State / Integration
**Linked Story:** US-001, US-002, US-003
**Estimate:** 2h
**Status:** Open

---

## Description

Set up state management (e.g., NgRx, Redux, Vuex, or context-based depending on frontend framework) to handle address data across components. Create actions, reducers, selectors, and effects (or equivalents) to fetch, create, update, and delete addresses. Integrate with backend APIs and manage loading/error states.

---

## Acceptance

- State module/store slice for addresses is created.
- API calls to address endpoints are encapsulated in service/effect classes.
- State holds: addresses list, current address (for editing), loading flags, error messages.
- Selectors provide derived data (e.g., getAddressById, getDefaultBilling, getDefaultShipping).
- Actions: loadAddresses, createAddress, updateAddress, deleteAddress, setDefaultBilling, setDefaultShipping.
- Components dispatch actions and subscribe to state changes.
- Proper error handling with user-friendly messages.
- Unit tests for state management logic (if appropriate for framework).
