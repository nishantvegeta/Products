**Feature:** Customer Address Management
**Layer:** API / Controller
**Linked Story:** US-001, US-002
**Estimate:** 1.5h

## Description

Implement API controller endpoints for address CRUD operations. Define routes following RESTful conventions and the project's API versioning strategy. Add permission authorization attributes to secure endpoints. Return responses in the standard ABP API response wrapper format.

---

## Acceptance

- AddressController with actions: POST (create), PUT (update), DELETE, GET (by id), GET (list/paged).
- Routes follow project convention: /api/app/addresses or similar.
- Each endpoint checks permissions using AbpAuthorize attribute (e.g., AddressManagement.Address.Create, etc.).
- Proper HTTP status codes returned (201 for create, 204 for delete, 200 for updates, 200/404 for get).
- Input validation returns 400 with error details when model validation fails.
- Controller injects and uses IAddressAppService.
- Swagger/OpenAPI documentation is generated automatically.
