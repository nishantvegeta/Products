## Description

Create Data Transfer Objects for address operations: CreateAddressDto, UpdateAddressDto, and AddressDto. Include all necessary fields with appropriate validation attributes. Follow ABP conventions for DTOs, including the use of ABP's validation attributes and the [Required] annotations for mandatory fields.

## Acceptance

- CreateAddressDto includes: Street, City, State, PostalCode, Country, and optional flags for defaultBilling, defaultShipping.
- UpdateAddressDto inherits from or mirrors CreateAddressDto but includes Id field.
- AddressDto is defined for read operations with all display fields.
- All DTOs have proper data annotations for validation (Required, StringLength, etc.).
- DTOs are placed in the correct application layer project (e.g., Acme.Products.Application.Contracts).
- Unit tests verify DTO validation rules work correctly.
