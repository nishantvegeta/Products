# T-018 — Create Frontend Page for Customer Address Management

**Feature:** Customer Address Management
**Layer:** Frontend — Page / View
**Linked Story:** US-001, US-002
**Estimate:** 2h
**Status:** Open

---

## Description

Build a customer-facing page where users can view their saved addresses, add new ones, edit existing ones, and set defaults. The page should include navigation (e.g., menu entry), proper layout using the project's UI framework (Blazor/Angular/React), and integrate with the address backend APIs. Use ABP LeptonX theme components where applicable.

---

## Acceptance

- Page accessible at a route like /account/addresses or /profile/addresses.
- List of saved addresses displayed with key fields (street, city, state, postal code, country) and default status indicators.
- Add Address button opens a form modal or navigates to an add page.
- Edit and Delete actions available per address (with confirmation for delete).
- Page respects customer ownership: only current user's addresses are shown.
- Responsive design works on desktop and mobile.
- Navigation menu includes link to Address Management under appropriate section (e.g., Profile).
- Basic loading and error states are handled.
