# T-011 — Extend Order DTOs to Include Address Information

**Feature:** Customer Address Management
**Layer:** Application Service
**Linked Story:** US-003, US-005
**Estimate:** 1h
**Status:** Open

---

## Description

Update Order-related DTOs to include billing and shipping address details. This may involve adding address fields directly to the OrderDto or including nested AddressDto objects. Ensure that both saved address references and ad-hoc address fields are captured appropriately for order display and processing.

---

## Acceptance

- OrderDto (and CreateOrderDto if separate) include either:
  - BillingAddressId and ShippingAddressId (for saved addresses), OR
  - Inline fields for billing/shipping addresses (for one-time entries).
- When an order uses a saved address, the DTO includes the address Id and possibly a populated AddressDto.
- DTOs support both Address scenarios per acceptance criteria.
- Mapping profiles are updated to handle the new address fields.
- Unit tests verify DTOs correctly represent orders with and without saved addresses.
