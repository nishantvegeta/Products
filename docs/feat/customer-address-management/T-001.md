# T-001 — Define Address Entity

**Feature:** Customer Address Management
**Layer:** Data / Entity
**Linked Story:** US-001
**Estimate:** 1h

---

## Description

Define the Address aggregate root entity with all required fields, relationships, and ABP base class. The entity must include: CustomerId (foreign key), address fields (street, city, state, postal code, country), optional phone, default flags for billing/shipping, and soft-delete support.

---

## Acceptance

- Address entity created with proper namespace and inherits from AggregateRoot (or Entity)
- Address entity includes Id (primary key), CustomerId, StreetLine1, StreetLine2 (optional), City, State, PostalCode, Country, Phone (optional), IsDefaultBilling, IsDefaultShipping, IsDeleted (ABP soft-delete), CreationTime, CreatorId, LastModificationTime, LastModifierId
- Address entity has a relationship to Customer (many-to-one)
- Entity is properly documented with XML comments
- Entity configuration uses ABP's EntityTypeConfiguration if needed for future
