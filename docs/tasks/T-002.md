## Technical Description

Implements the get address by ID endpoint. Must return the full AddressDto if the address exists and belongs to the authenticated customer, or return a 404 (via ResponseDataDto with Code=404) if not found. Must also check ownership: the address must belong to the authenticated customer.

---

## Approach

Add `GetAsync(Guid id)` method to `AddressAppService`. Follow the pattern from `CustomerAppService.GetAsync`:

1. Fetch the address by ID using `await _addressRepository.FindAsync(id);`
2. If null, log a warning and return `new ResponseDataDto<object> { Success = false, Message = "Address not found", Data = null, Code = 404 }`
3. Check that `address.CustomerId == CurrentUser.Id` (or session user ID). If not, return 404 or 403 depending on security approach — the spec says 403 for ownership, but for consistency with other endpoints you may return 404 to avoid information leakage. Check how CustomerAppService handles this (it doesn't appear to have ownership check in Get, so maybe ownership is enforced at query level?). Implement ownership check: if mismatch, return 404 or 403.
4. Map the entity to `AddressDto` using AutoMapper
5. Return `new ResponseDataDto<object> { Success = true, Message = "Address retrieved successfully", Data = dto, Code = 200 }`
6. Wrap in try-catch for exception handling as in CreateAsync

Apply `[Authorize(AddressPermissions.Default)]` attribute.
