## Technical Description

Implements the address list endpoint with filtering, pagination, and sorting. Must support the query parameters from the spec: `filter` (search by name/addressLine1/addressLine2/city/state/country), `isActive` (filter by active status with null meaning all), `maxResultCount`, `skipCount`, and `sorting`. Must enforce the default `isActive=true` when the parameter is not provided. Must return a paginated result with totalCount.

---

## Approach

Add `GetListAsync(GetAddressesInput input)` method to `AddressAppService`. Follow the pattern from `CustomerAppService` (if there's a list method) or from ABP's standard list pattern. Use `PagedResultDto<AddressDto>` as the response type inside the `ResponseDataDto<object>`.

Steps:

1. Get the current customer's user ID from `ICurrentUser` or `IAbpSession`
2. Build a query starting from `await _addressRepository.GetQueryableAsync()`
3. **Apply ownership filter**: `.Where(a => a.CustomerId == currentUserId)` — customers only see their own addresses
4. **Apply isActive filter**:
   - If `input.IsActive` has value (true or false), add `a.IsActive == input.IsActive`
   - If `input.IsActive` is null, do not filter (return both active and inactive)
   - Default behavior: spec says default is `true`, but `GetAddressesInput` has default `true`, so when client doesn't provide parameter, it will be true, so the filter will be `a.IsActive == true`. This matches spec.
5. **Apply text search on filter** (if `!string.IsNullOrWhiteSpace(input.Filter)`):
   - Search in Name, AddressLine1, AddressLine2, City, State, Country fields
   - Use `.Where(a => a.Name.Contains(filter) || a.AddressLine1.Contains(filter) || ...)` with EF.Functions.Like for case-insensitive if needed
   - Could also use ABP's `ISoftDelete` if applicable; but simpler to chain multiple conditions
6. **Apply sorting**:
   - If `input.Sorting` is provided, use `.OrderBy(input.Sorting)` (but be careful: EF Core may not support string-based OrderBy directly; safer to parse the sorting string and apply specific properties)
   - Alternative: use ABP's `ApplyOrderBy` extension if available, or implement simple switch: if sorting=="name asc" then `.OrderBy(a => a.Name)`, if "name desc" then `.OrderByDescending(a => a.Name), etc. Start with default `"name asc"`
   - For simplicity, default sorting to `a.Name` ascending
7. **Apply pagination**: Use `await AsyncExecuter.ToListAsync(query.Skip(input.SkipCount).Take(input.MaxResultCount))` — remember to use ABP's `AsyncExecuter` for IQueryable async operations (see CustomerAppService usage)
8. **Get total count**: Before pagination, get `var totalCount = await AsyncExecuter.CountAsync(query)` (without Skip/Take)
9. **Map to DTO**: Map the list of entities to `List<AddressDto>` using AutoMapper: `_mapper.Map<List<Address>, List<AddressDto>>(addresses)`
10. **Return**: `new ResponseDataDto<object> { Success = true, Data = new { totalCount, items = addressDtos }, Code = 200 }`

Note: The response wraps `{ totalCount, items }` inside the `data` field as a simple object, not using `PagedResultDto` wrapper inside. Check spec: it says Response: 200 OK with `{ totalCount, items }`. So return that anonymous object structure.

Apply `[Authorize(AddressPermissions.Default)]` attribute.
