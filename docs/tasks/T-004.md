## Technical Description

Implements the address update endpoint. All fields in UpdateAddressDto are optional — only provided fields should be updated. Must enforce validation rules and handle the business rule that only one default billing/shipping address can exist per customer: if updating an address to set IsDefaultBilling=true, all other addresses for that customer with IsDefaultBilling=true must be set to false. Same for IsDefaultShipping.

---

## Approach

Add `UpdateAsync(Guid id, UpdateAddressDto input)` to `AddressAppService`. Follow CustomerAppService.UpdateAsync pattern:

1. Validate input via `ValidateInputAsync(input, id)`
   - Check string length constraints for any provided string fields (if provided)
   - Verify customer ownership (address must belong to current user)
2. Fetch the address by id: `var address = await _addressRepository.FindAsync(id);`
   - If null, return 404 ResponseDataDto
   - Check `address.CustomerId == CurrentUser.Id` — if not, return 404 or 403
3. Use AutoMapper to ** selectively ** update: ` _mapper.Map(input, address); ` This will update only non-null properties if configured correctly. Check if AutoMapper is set up to ignore null values globally or per profile. If not, manual property-by-property update is needed:
   ```csharp
   if (input.Name != null) address.Name = input.Name;
   if (input.AddressLine1 != null) address.AddressLine1 = input.AddressLine1;
   // etc.
   ```
   Prefer the existing AutoMapper configuration style; check existing Update methods to see how they handle partial updates. CustomerAppService uses ` _mapper.Map(input, customer); ` — so AutoMapper is configured to map onto existing entity without overwriting with nulls. Verify this behavior in AutoMapper profile configuration (likely uses `ForAllMembers(opts => opts.Condition(...))`). Follow that.
4. Handle default address business rules **before** saving:
   - If `input.IsDefaultBilling == true`, query all other addresses for this customer where `IsDefaultBilling == true && id != address.Id`, and set them to false (loop and update using repository)
   - If `input.IsDefaultShipping == true`, same for IsDefaultShipping
5. Save changes: `await _addressRepository.UpdateAsync(address);` This will also cascade to modified other addresses if any were updated in step 4. Ensure those are also updated.
6. Map the updated entity to `AddressDto`
7. Return success ResponseDataDto with code 200

Apply `[Authorize(AddressPermissions.Edit)]`.
